# /verify - 기능 개발 후 자동 검증 및 리포트

기능 개발이 완료되면 이 명령을 실행하여 전체 검증을 수행합니다.

## 검증 단계

순서대로 실행하고, 각 단계의 결과를 기록합니다:

### 1. TypeScript 타입 체크
```bash
npm run typecheck
```

### 2. ESLint 린트
```bash
npm run lint
```

### 3. 빌드
```bash
npm run build
```

### 4. 단위 테스트 (Vitest)
```bash
npm run test
```

### 5. E2E 테스트 (Playwright) - 선택적
```bash
npm run test:e2e
```

## 검증 프로세스

1. **각 단계를 순차적으로 실행**
2. **실패 시**:
   - 에러 메시지 분석
   - 1회 자동 수정 시도
   - 재검증
   - 여전히 실패하면 리포트 생성
3. **성공 시**: 다음 단계로 진행

## 자동 수정 규칙

| 에러 유형 | 수정 방법 |
|----------|----------|
| 타입 에러 | 누락된 import/타입 추가 |
| 린트 에러 | `npm run lint -- --fix` 실행 |
| 빌드 에러 | import 경로, export 수정 |
| 테스트 실패 | 테스트 코드 또는 구현 수정 |

## 리포트 형식

검증 완료 후 다음 형식으로 리포트를 출력하세요:

```
## 🔍 검증 결과 리포트

### 📊 요약
| 단계 | 상태 | 소요 시간 |
|------|------|----------|
| 타입체크 | ✅/❌ | Xs |
| 린트 | ✅/❌ | Xs |
| 빌드 | ✅/❌ | Xs |
| 단위 테스트 | ✅/❌ | Xs |
| E2E 테스트 | ✅/❌/⏭️ | Xs |

### 🐛 발견된 문제 (실패 시)
- [파일:라인] 에러 메시지
- ...

### 🔧 수정 시도
- [수정한 내용]
- [결과: 성공/실패]

### 📋 권장 조치 (수동 개입 필요 시)
1. ...
2. ...

### ✅ 최종 결과
- 전체 통과 / 부분 실패 / 수동 개입 필요
```

## 실행 지침

1. 위 단계들을 순서대로 실행
2. 각 단계의 출력을 캡처
3. 에러 발생 시 분석 후 1회 수정 시도
4. 최종 리포트 생성

**중요**: E2E 테스트(`npm run test:e2e`)는 개발 서버가 필요하므로, 시간이 오래 걸리거나 서버가 없으면 스킵해도 됩니다.
