# FesMate UX 구조 설계

## 📱 네비게이션 구조

### 데스크톱 - 상단 메뉴
```
[Main] [이벤트허브] [양도허브] [아티스트허브] [마이로그]
```

### 모바일 - 하단 탭 네비게이션 (토스 앱 스타일)
```
┌─────────────────────────────────┐
│                                  │
│        (메인 콘텐츠 영역)         │
│                                  │
└─────────────────────────────────┘
[🏠 홈] [🎪 이벤트허브] [📝 양도] [⭐ 아티스트] [👤 MY]
```

---

## 🗺️ 라우팅 구조

```
/                         - Main (홈)
/events/hub               - 이벤트허브 메인 (= 공연목록)
/events/[id]              - 특정 이벤트 상세 페이지
/events/[id]/timetable    - 스마트 타임테이블 전체 페이지
/companions               - 양도허브 메인
/companions/[id]          - 특정 양도글 상세
/artists                  - 아티스트허브 메인
/artists/[id]             - 특정 아티스트 상세
/my                       - 마이로그
/my/gonglogs              - 나의 공연 기록
/my/tickets               - 나의 티켓북
```

**핵심 변경점**:
- `/events` 라우트 제거 → `/events/hub`가 공연목록 역할
- 공연목록 = 이벤트허브 메인 (동일한 페이지)

- **날짜**: 전체 / 오늘 / 이번주 / 이번달 / 직접선택

- **공연타입**: 콘서트 / 페스티벌 / 전시 / 뮤지컬 / 클래식
- **가격대**: 전체 / 무료 / ~5만원 / 5~10만원 / 10만원~

#### 핵심 기능

**타임테이블 (간략 버전)**
- 현재 시간 기준 앞뒤 1개씩 표시
- ▶ 마크로 현재 진행 중 표시
- [전체 타임테이블 보기] 클릭 → `/events/[id]/timetable`로 이동

**아티스트 섹션**
- **웹**: 아티스트별로 호응법/셋리스트 전부 표시, 많으면 [다음 아티스트] 버튼으로 추가 로드
- **모바일**: 아티스트 프로필 이미지만 표시, 클릭 시 `/artists/[id]` 상세 페이지로 이동

**현장 제보**
- 1차 필터: 타입별 탭 (줄상황 | MD재고 | 시설 | 긴급)
- 2차 정렬: 최신순 / 신뢰도순
- GPS 위치 검증 (공연장 반경 1km)

**양도 게시판**
- 이 공연에 해당하는 양도글만 필터링
- [+ 양도글 작성] 시 공연 정보 자동 입력

### 4. 스마트 타임테이블 페이지 (`/events/[id]/timetable`)

#### 레이아웃 (전체 타임테이블)
```
┌─────────────────────────────────┐
│  ← 뒤로    스마트 타임테이블     │
├─────────────────────────────────┤
│  날짜: 2025-12-31               │
│  [내보내기 (ICS)]                │
├─────────────────────────────────┤
│  ⏰ 전체 타임라인                │
│                                  │
│  18:00  게이트 오픈              │
│  ⭐ 45% 체크                    │
│                                  │
│  ┌─────────────────────────┐    │
│  │ 19:00  아티스트 A        │ ← 클릭 가능│
│  │ ☑️ 체크 (알림 받기)      │    │
│  │ 이용자의 62%가 체크      │    │
│  └─────────────────────────┘    │
│  ▶ Now Playing              │
│                                  │
│  ┌─────────────────────────┐    │
│  │ 20:00  아티스트 B        │    │
│  │ □ 체크                   │    │
│  └─────────────────────────┘    │
│                                  │
│  21:00  인터미션                 │
│                                  │
│  ┌─────────────────────────┐    │
│  │ 21:30  아티스트 C        │    │
│  │ □ 체크                   │    │
│  └─────────────────────────┘    │
│                                  │
│  23:00  종료                     │
└─────────────────────────────────┘
```

#### 기능
- **⭐ 체크 기능**: 원하는 아티스트에 체크 → 10분 전 알림
- **통계 표시**: "이용자의 X%가 체크"
- **Now Playing**: 현재 진행 중인 순서 강조
- **아티스트 클릭**: 해당 아티스트 상세 페이지 (`/artists/[id]`)로 이동
- **ICS 내보내기**: 개인 캘린더(구글/애플)에 동기화

## 🎨 디자인 가이드라인

### 색상 팔레트 (예시)
- **Primary**: 보라/핑크 그라데이션 (공연 느낌)
- **Secondary**: 네온 블루/그린 (실시간 강조)
- **Accent**: 오렌지/레드 (긴급/중요)
- **Neutral**: 회색 계열 (텍스트, 배경)

### 타이포그래피
- **헤드라인**: Bold, 20-28px
- **본문**: Regular, 14-16px
- **캡션**: 12px

### 컴포넌트 스타일
- **카드**: 둥근 모서리 (12px), 그림자 효과
- **버튼**: 명확한 CTA (Pill 형태)
- **배지**: 작고 눈에 띄는 라벨 (🔴 Live, D-day)

## 🔄 인터랙션 패턴

### 로딩 상태
- 스켈레톤 UI (포스터, 텍스트 라인)
- 스피너는 최소한으로

### 실시간 업데이트
- 새 제보 시 상단에 "새 제보 1건" 배너 → 클릭 시 새로고침
- Socket 연결 끊김 시 "실시간 연결 끊김" 안내

### 빈 상태 (Empty State)
- "아직 등록된 공연이 없어요" + [공연 등록하기] 버튼
- 일러스트 추가

### 에러 상태
- 친근한 메시지 + 재시도 버튼
- 네트워크 오류, 404, 500 등 구분
