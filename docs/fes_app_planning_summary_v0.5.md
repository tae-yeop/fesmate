# 문화 행사 앱 기획 정리 (진행 중 초안) v0.5

> 범위: 공연/전시/페스티벌을 포함하는 **행사(Event)** 중심 서비스  
> 전제: **써드파티 정보/커뮤니티 서비스** (예매처 좌석/재고 실시간 연동 불가, 직접 판매/안전결제/에스크로 없음)

---

## 용어 정의 (Glossary)

- **행사(Event)**: 공연/전시/페스티벌 등 “일정(시작/종료)”과 “장소”를 갖는 최상위 엔터티.
- **행사 페이지(`/event/{id}`)**: 행사 상세 + 허브 + 타임테이블 + 아티스트를 한 라우트 안에서 탭으로 제공하는 화면.
- **허브(Hub)**: 행사 안에서 공지/실시간 제보/커뮤니티/리뷰·영상을 통합해 보는 피드 및 요약 영역.
- **LIVE 모드**: 행사 진행 전/중에 맞춰 “실시간(게이트/MD/시설/안전) + 타임테이블”을 최우선으로 보여주는 허브 상태.
- **RECAP 모드**: 행사 종료 후 회고 목적의 허브 상태. “총평/베스트 후기·영상/슬롯 요약”을 최우선으로 보여준다.
- **⭐찜(Wishlist)**: 관심/예매 예정/갈 수도 있는 행사 저장 상태.
- **✅다녀옴(Attended)**: 관람 완료 상태. 리뷰/영상/기록 기능의 기준이 되는 상태.
- **슬롯(Slot)**: 타임테이블의 “특정 시간·스테이지(또는 전시 세션)·아티스트(또는 프로그램)” 단위.
- **Now/Next**: 타임테이블에서 현재 진행/다음 순서를 요약해 보여주는 영역.
- **글 타입(Post Type)**: 허브/커뮤니티에서 글을 구조화하기 위한 분류(예: 게이트, MD, 시설, 안전, 동행, 양도, 후기, 영상, 질문·팁).
- **템플릿(Template)**: 글 타입별 입력 폼(필수/선택 필드)을 미리 정의한 작성 방식.
- **UGC**: 사용자 생성 콘텐츠(제보/후기/영상/커뮤니티 글).
- **Official 공지**: 운영자 또는 공식 채널 기반으로 등록되는 공지(링크/텍스트 중심).
- **도움됨(Helpful) 반응**: 글 품질을 신호로 삼는 피드백(정렬/요약/신뢰도에 활용).
- **신뢰도(A/B/C)**: 실시간 제보의 품질을 단순 등급화한 표시(최근성/증빙/작성자 신호 등 기반, MVP는 단순 규칙).
- **MyFes**: 사용자의 ⭐찜/✅다녀옴을 섞어 “예정+지난”을 한 타임라인으로 관리하는 개인 아카이브 탭.

- (MVP) **목업/시드 데이터 + Dev 메뉴(디버그)**로 UI/UX와 핵심 플로우를 빠르게 검증한다.

## 3) 핵심 상태 모델: ⭐찜 vs ✅다녀옴
### 정의
- ⭐ **찜(Wishlist)**: 관심/예매 예정/갈 수도 있음
- ✅ **다녀옴(Attended)**: 실제 관람 완료 (리뷰/영상/기록의 기준)

### 노출(표시/조작) 원칙 — “3곳”
1) **행사 상세 헤더(조작 메인)**: ⭐찜 / ✅다녀옴 버튼 노출  
2) **탐색 카드(표시 + ⭐빠른 토글)**: 카드에 ⭐/✅ 배지(✅는 표시 위주)  
3) **MyFes 타임라인(구조로 반영)**: 찜/다녀옴/진행중 상태가 카드와 동선에 반영

### 권장 UX 룰
- ✅다녀옴은 행사 당일~종료 후 일정 기간에 활성(또는 종료 후 RECAP 진입 시 1회 제안)
- ✅다녀옴을 누르면 기본적으로 ⭐찜도 함께 켜짐(옵션)

---

## 4) 탐색(행사) — 3가지 뷰
> 뷰를 바꿔도 **검색/필터/정렬 상태 유지**

### A) 카드뷰 (브라우징)
- 그리드 카드로 여러 행사 빠르게 탐색
- 카드에 ⭐찜 토글, ✅다녀옴 배지(표시), 진행중 배지

### B) 리스트뷰 (정보 밀도)
- 빠르게 많이 훑기 좋은 리스트
- 정렬/필터와 궁합 좋음

### C) 캘린더뷰 (계획)
- 캘린더 날짜 클릭 → **대표 카드 1장** 우선 노출(패널/바텀시트)
- 대표 카드 선정: **내 찜/팔로우 기반 우선**, 나머지는 스크롤/더보기로 확인
- 날짜 셀에 “행사 있음” 표시(점/숫자)

### 지난 행사 탐색 범위
- **최근 1~2년**(기본 12개월, 최대 24개월)
- 클릭 시: 해당 행사는 **RECAP 모드**로 진입

---

## 5) 행사 페이지: 상세 + 허브는 “한 페이지 내부 탭/모드”
> 행사 상세와 허브를 별도 페이지로 분리하지 않고, **/event/{id}** 내부 탭으로 운영

### 행사 상세 상단 헤더
- ⭐찜 / ✅다녀옴
- 예매 일정/링크/리마인더(좌석 잔여/공식 재고 실시간 연동은 없음)

### 권장 탭 구성
- **개요**
- **허브(라이브/RECAP)**
- **타임테이블**
- **아티스트**
- (선택, P2) **후기·영상**(허브 피드의 후기/영상 필터 뷰)

---

## 6) 행사 허브(핵심): 4박스 요약 + 허브 피드
### A) 상단 4박스(요약)
1) **실시간**: 게이트/MD/시설/안전 요약(신뢰도/업데이트 시각)  
2) **타임테이블**: 현재/다음 아티스트, 시간, 진행 바  
3) **공식안내**: Official 공지(핀 고정), 인스타 링크/미리보기(가능 범위)  
4) **커뮤니티 요약**: 동행/양도 최신 글 미리보기

### B) 허브 피드(모든 글 타입이 한 곳에)
- 필터 칩(글 타입):  
  - 실시간(게이트/MD/시설/안전) / 타임테이블 관련 / 공식 / 동행 / 양도 / **후기** / **영상** / 질문·팁
- 하단 고정 CTA **+ 올리기** → 타입 선택 → 템플릿 작성
- 정렬(기본):
  - **LIVE**: 최신(최근성) 우선 + 도움됨/증빙(사진) 가중(옵션: 도움됨)
  - **RECAP**: 도움됨/평점 우선 + 최신 보조(옵션: 최신)

### C) 허브 모드 전환
- **LIVE 모드(진행 전/중)**: 실시간·타임테이블 우선, 후기/영상은 필터로 접근
- **RECAP 모드(종료 후)**: 총평/베스트 후기·영상/아티스트 요약이 상단으로 전환

### D) LIVE → RECAP 전환 규칙 (MVP 확정)
- 기준: 행사 로컬 타임존(기본 Asia/Seoul)
- 기본 LIVE: `현재 >= (start_at - 24h)` AND `현재 < (end_at + 6h)`
- 기본 RECAP: `현재 >= (end_at + 6h)`
- 예외: 운영자 `override_mode(LIVE/RECAP/AUTO)`로 강제 가능

## 7) 후기/영상 기능 (행사 허브 내부 “글 타입”)
### 후기(리뷰) — 2레벨
1) **행사(페스티벌) 총평 리뷰(별점 + 텍스트)**  
2) **아티스트 리뷰는 “이 행사에서의 무대(슬롯)” 귀속** (별점 + 텍스트)

### 영상 공유
- MVP 권장: **외부 링크 공유(유튜브/인스타/틱톡 등)** + 미리보기
- 선택 메타: 아티스트/스테이지/시간대(타임테이블 슬롯 연결)

### 권한/품질
- 총평/슬롯 리뷰는 원칙적으로 ✅다녀옴 기반에서 작성 유도
- 페스티벌은 “해당 슬롯 봤어요(👀)” 같은 토글로 정밀도 강화(선택)

---

## 8) 아티스트 정보
- 아티스트 페이지: 프로필/팔로우, **호응법**, **셋리스트(회차별)**, 행사별 슬롯 리뷰 모아보기
- 타임테이블에서 아티스트 클릭 → 아티스트 정보(호응법/셋리)로 빠르게 이동(드로어/모달 권장)

---

## 9) 커뮤니티(큰 묶음)
- 커뮤니티 내부 카테고리:  
  - 동행(일반) / 택시팟 / 밥 / 숙소 / 직거래양도 / 후기·팁 / 질문
- 상단 기본 필터: 내 MyFes(다가오는 행사) 또는 현재 보고 있는 행사
- 템플릿 글쓰기(시간/장소/인원/예산/규칙/연락 방식)
- 자동 만료: 카테고리별 기준 시각 기반(예: 택시=출발+2h, 밥=만남+3h 등) → 상세는 UX/IA 참조

> 결제/에스크로는 제공하지 않음. 대신 최소 안전장치(연락처 마스킹, 신고/차단, 가이드라인/경고) 중심.

---

## 10) MyFes: 혼합 타임라인(예정+지난을 한 줄로)
### 핵심 결정
- **예정+지난을 한 타임라인에 섞어서** 일자별로 쭉 표시
- 기본 진입 위치: **오늘 근처** (오늘 앵커 + “오늘로 이동”)

### 타임라인 카드 요소
- 상태 배지: ⭐찜 / ✅다녀옴 / 진행중
- 퀵 액션:
  - 예정/진행중: 허브, 내 타임테이블
  - 다녀옴: RECAP 보기, 총평/슬롯 리뷰, 영상 추가
- 부가: 내가 남긴 것 카운트(리뷰/영상/제보)

## 11) 제약/전제(명시)
- 예매처 좌석 잔여/공식 재고 등 **실시간 공식 데이터 직접 연동 불가**  
  → 예매 일정/링크/리마인더 + 커뮤니티 제보(신뢰도/최근성 표시)로 대체
- 직접 판매/결제/안전결제/에스크로 **미제공**

- (MVP) 행사 데이터는 **목업/시드 데이터**로 제공하여 UI/UX 검증을 우선한다.
- (Future) 예매처/주관사 최신 공지(웹) + SNS(인스타/트위터 등)를 주기적으로 수집→표준화하고,
  변경 감지 시 제안→승인/반영(+필요 알림) 워크플로우로 확장.
- 로그인: **카카오/네이버/구글**. 비로그인은 조회 가능, 작성/댓글/⭐찜/✅다녀옴/알림 설정은 로그인 필요.
- (MVP) 별도 Admin 도구는 **미구현**. 대신 **Dev 메뉴(디버그)**로 시간/상태/override/알림을 시뮬레이션한다.
- (Future) 운영자(관리자) 도구: 허브(Official 공지/핀/요약) 편집, LIVE/RECAP override, 행사 상태(연기/취소) 변경, 신고 처리.
