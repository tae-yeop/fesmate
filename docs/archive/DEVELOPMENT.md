페스메이트 (FesMate)

# 0. 개발 동기
- 콘서트나 페스티벌, 전시회 등 문화 활동을 하는 사람들을 위한 편의 및 커뮤니티 서비스가 산발적으로 흩어져 있음
- 캘린더나 타임테이블, 예매 소식 정보, 동행이나 양도 게시글 등록 등 공연 및 전시를 가는 사람들에게 필요한 기능들을 통합해서 제공.
- 목표: 공연/페스티벌/전시 현장의 모든 정보를 한 화면(허브)에 모으고, 기록·동행·랭킹까지 이어지는 팬 경험 OS 구축.
- 랭킹 리더보드 등 서비스만의 특화된 기능을 제공하여 계속 서비스를 사용할 수 있도록 함



# 1. 코어 가치 & 페르소나

코어 가치: 내가 다녀온/가고 싶은 공연을 기록·공유하고, 예매 타이밍과 일정 충돌을 똑똑하게 관리하며, 안전하게 동행/양도 커뮤니티를 즐긴다.

페르소나: 
- (a) 헤비팬/페스티벌러: 줄/재고/셋리스트/호응법을 빠르게 확인하고 친구와 움직인다.
- (b) 입문자: 추천과 가이드를 통해 공연 정보를 얻는다
- (c) 안전한 양도/동행이 필요한 이용자: 택시·식사·숙소 동행을 안전하게 모집/참여한다
- (d) 주최/프로모터: 공지와 재고/혼잡 업데이트를 공식 레이어로 배포


# 2. 반드시 넣을 코어 기능 (MVP)

- 신규 공연 정보 제공
    - 자체 크롤링
    - 사용자가 직접 이벤트 등록
    - 예매 이벤트 알림: 이벤트별 예매 오픈/마감, 좌석 추가, 라인업 변경 등의 공지사항 실시간 푸시 (사용자가 관심을 표시한 공연에 대해서)

- 공연 허브 : 여러 사람이 만드는 특정 공연 관련 페이지
    - 공연 사진, 영상 업로드 가능
    - 실시간 정보 공유
        - 게이트 상황
        - MD 상황
        - 시설 현황
    - 타임테이블 (공연 제공 타임 테이블)
    - 안내: 공연 관련
    - 같이해요! : 택시팟, 밥, 숙소 등 동행 찾는 게시판
    - 아티스트 정보: 셋리스트 필수 제공
    - 관련 팁 공유
        - 맛집 공유


- 스마트 타임테이블
    - 개인 캘린더(ICS) 동기화
    - 통계 제공: 이용자의 몇프로는 이걸 체크했다
    - 친구, 동행 레이어 겹쳐보기


- 구조화된 동행/양도 폼
    - 티켓 양도
    - 숙소 양도
    - 택시팟 양도
    - 날짜·집합 장소·예산 분담·역할(운전/사진/초행)
    - 약속 방식까지 템플릿으로 1분 등록
    - 구매 관심있는 사람은 리스트로 등록되어 판매자에게 보여짐


- 공연 후기 기능
    - 공연/공연장/좌석블록별 후기(가시성·사운드·대기열 팁) + 사진 증빙 태그.


- 개인 공연로그 (GongLog)
    - “연대기(타임라인) + 통계” 
    - 아티스트/장르/도시/동행 횟수, 사진/세트리스트 첨부.


# 3. 차별화 포인트(리텐션 장치)

- 연말 결산/배지(POAP 느낌)
    - “2025 나의 공연 리포트”, 출석 배지, 지역·장르 배지. 프로필에 표시
- 예매 어시스트
    - 친구가 관심표시한 티켓 알림
- 날씨·체크리스트
    - 당일 우천/한파 시 자동 체크리스트(우비, 보조배터리 등) + 픽업 리스트 공유
- 친구/FOMO 루프
    - 친구가 예매/후기/동행 모집 시 피드 상위 노출
    - “함께 가기” 한 탭 초대
- 리더보드
    - 도움되는 후기’/‘질문 답변’/‘신규 행사 제보’ 점수화. **윌슨 스코어** 등으로 추천 농락 방지, 최근성 가중치.
    - 년도별 시즌화


# 4. 신뢰/안전(필수)
- 본인확인 단계적 적용: 인증(휴대폰/간편인증)
- 모임 안전 옵션: 공개/비공개, 참가 승인, 성비/연령 범위, 가이드라인 체크박스
- 거래 안전장치: 사용자 랭킹 점수를 통해 신뢰도 파악. 직거래만 가능


# 5. 커뮤니티 & 소셜

- 클럽/크루: 지역·장르 크루 만들기, 정기 모임, 캘린더 공동 관리
- 콘텐츠 포맷: 질문/리뷰/팁/양도/동행 등 **포스트 타입**을 분리해 피드가 깔끔하게.
- 나의 티켓북
    - 실제 티켓 등록하기 → 사진 촬영 → ai 이미지 자동 편집 (개인정보는 가리고) 번호랑 예매 번호만 나오게 → 조명 보정후 나의 티켓북에 등록됨
    - SNS/Kakao 공유

# 6. 추천 & 개인화(ML로 차별화)

- **개인 취향 벡터**: 다녀온 공연/팔로우 아티스트/리액션으로 임베딩 → **콘텐츠·동행·행사**를 통합 추천.
- **충돌 회피 추천**: 같은 주의 여러 공연 중 우선순위 제안(거리·친구 참석·가격·후기 품질).
- **탐색 모드**: 새로 뜨는 소극장/인디 라인업 “발견 피드”

# 7. 주최/아티스트 도구(B2B 루프)

- 추후 서비스가 성장하여 주최 업체와 협업 가능
- 오거나이저 대시보드: 라인업/공지/타임테이블 업데이트 → 팔로워 푸시.
- 부스/스폰서 맵: 쿠폰/미션(스탬프 투어) 발급, 실시간 참여 데이터.
- UGC 리퀘스트: 후기 리그램/인용 허가 요청 워크플로.

# 10. 기술 스택
- 클라이언트: React Native(모바일앱) + Expo EAS / 웹은 Next.js(PWA). 디자인 시스템 Tailwind(웹), RN Paper/NativeWind.
- 실시간: WebSocket/Socket.IO(or GraphQL Subscriptions), 로컬 큐(오프라인 제출)
- 백엔드: Node.js(NestJS) or Python(FastAPI), GraphQL + REST 혼용(웹훅/업로드는 REST), OpenAPI 문서화
- DB/검색: PostgreSQL(+PostGIS), Redis(캐시/큐), OpenSearch(검색/랭킹). S3 호환 스토리지(이미지/오디오클립).
- 지도/로그인/푸시: 카카오/네이버 지도 딥링크, 카카오/네이버/애플 로그인, FCM/APNs.
- 분석/로그: PostHog/Amplitude, Sentry/Datadog, BigQuery(옵션).
