행사(event) 허브 페이지 디테일

행사는 다음과 같은 타입으로 구성.
각 타입마다 세부 메뉴 차이가 있을 수 있음
- 공연: 게스트, 메인 아티스트, 여러일 가능
- 페스티벌: 여러일, 다수 아티스트, 타임테이블 지원
- 전시: 전시 내용
- 뮤지컬: 출연진 정보
- 클래식: 출연진 정보, 레파토리


- 공연 진행 중 필요한 정보(줄/재고/입장/화장실/굿즈 위치 등)를 **참가자들이 즉시 올리고, 신뢰도 기반으로 집계**해 한 화면에 보여주는 페이지

## 화면 레이아웃


### 리포트 타입 & 입력 폼(구조화)

- QueueReport : `장소(게이트A/부스명)`, `현재 대기열(명/대략)`, `내 위치까지 걸린 시간`, `사진`.
- **MerchStockReport :** `부스`, `아이템`, `사이즈`, `재고수준{많음/보통/적음/품절}`, `가격(선택)`, `사진`
    - MD 재고 목록을 넣어서 가능, 품절인지 투표할수 있게
    - 어뷰즈할 수 있을거 같은데 어떻게 해야할까? 영수증 인증?
    - 사진 첨부 상황을 보고 vlm이 판단해야하나?
- FacilityReport : `화장실/물/휴게`, `혼잡{여유/보통/혼잡}`, `청결/비품`.
- **Safety/Alert**: 미끄럼/넘어짐/분실/응급/분쟁 등 프리셋 + 위치
    - 분실물 중 개인 정보가 있는건 지울 수 있게 간단한 이미지 편집도구 필요 (신분증 같은거)

### 화면 구성

- **허브 4박스 레이아웃(모바일 기준)**
    - 상단 고정 헤더: 이벤트명·회차·현재 시각·날씨/혼잡 배지 · 탭(개요｜맵｜타임테이블｜공지｜현장제보)
    - 본문 2×2 그리드(스크롤): **실시간**, **타임테이블**, **공식안내**, **동행**
    - 하단 고정: [+ 제보] / [아티스트] / [내 일정] 퀵액션

A. 실시간 박스(현장 제보 요약)


# 아티스트 페이지

## 기본 구성

- 헤더: 이름/사진/장르/공식 링크/팔로우
- 호응법(콜&리스폰스)
    - 곡별 카드: 후렴 파트 라인별 스크립트(한글/로마자) + 미니 오디오(5–10초)
    - 타이밍 가이드: “1절 0:48~1:12, 후렴 진입 직전 박수 4박”
    - 현장 매너: 점프/떼창/조용 관람 구간, 플래시 금지 등
- 셋리스트
    - 회차별 탭: “2025-10-31 서울” (커뮤니티 제보 기반, 신뢰도 등급 표시)
    - 곡별 링크 → 곡 정보/응원법 카드로 점프
- 해당 공연 멘트
    - 공연에서 어떤 멘트를 했는지 집단 지성 wiki 형태로 편집

## 허브와의 연결 UX
- 타임테이블에서 ‘i’/‘호응법’/‘셋리스트’ 탭 → 아티스트 페이지의 해당 앵커로 이동(드로어/모달)
- 공연 중엔 미니 호응 가이드 바를 상단에 띄움(현재 곡 추정 시각 구간에 맞춰 한 줄 강조)
- 타임테이블에서 ⭐한 아티스트는 공연 10분 전에 “호응법 빠른보기” 푸시/인앱 배너
